#### 第 1 节：能跑起来的第一个 Shader（纯色+渐变+时间动画）
- **目标效果**：一个平面/精灵，颜色随时间在两种颜色之间来回变化，带水平渐变。  
- **核心知识点**：  
  - Effect 文件基本结构（`techniques` / `passes` / `properties` / `CCProgram`）。  
  - 顶点着色器 VS / 像素着色器 FS 输入输出（position/uv）。  
  - uniform/property 与材质参数映射。  
  - 在 TS 里通过 `setProperty` 更新 uniform。  
- **交付物**：  
  - 1 个 `.effect` 文件。  
  - 1 个使用该 effect 的 Material。  
  - 1 个场景截图（能看到颜色随时间变化）。  
- **失败自检点**：黑屏 / 纯黑纯白 / 颜色不变 / 控制台报 shader 编译错误。

#### 第 2 节：UV 基础与滚动（UV Scroll）
- **目标效果**：一张贴图在 X 方向循环滚动（水流/传送带效果）。  
- **核心知识点**：  
  - UV 坐标系与 `a_uv0`、`v_uv`。  
  - 使用时间偏移 UV (`uv + speed * time`)。  
  - 纹理采样 `texture` / `CCTexture2D`。  
- **交付物**：UV 滚动特效截图 + 可调滚动速度参数。  
- **自检**：滚动方向/速度可控；无锯齿或明显断裂。

#### 第 3 节：遮罩 & 渐变（Mask / Gradient）
- **目标效果**：基于 UV 的渐变遮罩（上下/左右淡入淡出），支持调节过渡宽度。  
- **核心知识点**：  
  - 使用 `smoothstep` 做平滑边缘。  
  - 使用一张灰度 Mask 纹理控制透明度。  
  - Blend 基础（透明叠加前提）。  
- **交付物**：可调渐变方向/宽度的遮罩特效。  
- **自检**：边缘是否硬/锯齿；透明区域是否真的不渲染。

#### 第 4 节：溶解（Dissolve）
- **目标效果**：一张图按噪声纹理从某个阈值开始溶解，有火焰/描边效果。  
- **核心知识点**：  
  - 利用噪声纹理 + 阈值控制溶解进度。  
  - 使用 `step` / `smoothstep` 与 UV 组合。  
  - 边缘颜色 / 描边（基于阈值附近的区间）。  
- **交付物**：一套可控溶解特效（含进度与边缘色）。  
- **自检**：阈值从 0→1 时，溶解从无到全；边缘是否连续。

#### 第 5 节：基本光照（Lambert / Blinn-Phong 入门）
- **目标效果**：一个简单 3D 物体（球/盒子）受单向光照，能看到明暗变化。  
- **核心知识点**：  
  - 法线（normal）与 `dot(N, L)`。  
  - 世界空间/视空间转换。  
  - 使用 Cocos 内置的 light/uniform。  
- **交付物**：一个自定义光照 Shader（取代内置 unlit）。  
- **自检**：转动物体或光源时，高光/暗部是否跟着移动。

#### 第 6 节：法线贴图（Normal Map）
- **目标效果**：在平面上通过法线贴图伪造凹凸感。  
- **核心知识点**：  
  - 切线空间（Tangent Space）基本概念。  
  - 从 normal map 解码法线。  
  - 组合主纹理 + 法线贴图。  
- **交付物**：同一张贴图，有/无法线贴图前后对比截图。  
- **自检**：灯光方向改变时，凹凸细节是否随之变化。

#### 第 7 节：描边（Outline）
- **目标效果**：3D 模型有单色描边（卡通渲染基础）。  
- **核心知识点**：  
  - 多 Pass：膨胀模型背面实现轮廓线。  
  - 深度测试/写入控制。  
  - Pass 顺序与渲染结果。  
- **交付物**：带描边的角色/物体截图。  
- **自检**：从不同角度看，描边是否稳定且不穿帮。

#### 第 8 节：水波/扭曲（Distortion）
- **目标效果**：水面或热浪扭曲背景画面。  
- **核心知识点**：  
  - 使用法线/噪声纹理偏移 UV 实现扭曲。  
  - 时间驱动波动。  
  - 精度与采样次数对性能的影响。  
- **交付物**：一块「会动」的水面或空气扭曲特效。  
- **自检**：扭曲方向是否自洽；是否出现明显断层或破图。

#### 第 9 节：屏幕后处理（Post-Processing 基础）
- **目标效果**：简单的全屏特效（灰度、色偏、渐变 vignette 等）。  
- **核心知识点**：  
  - 使用 camera + RenderTexture 实现后处理。  
  - 全屏三角形 / quad 渲染。  
  - 颜色空间（sRGB/Linear）基础注意点。  
- **交付物**：至少 1 个可开关的全屏后处理特效。  
- **自检**：开关后处理时，只有颜色变化，无布局/尺寸异常。

#### 第 10 节：混合(Blend) / 深度(Depth) / 模板(Stencil) 控制
- **目标效果**：
  - 半透明叠加、加法发光、写入/不写入深度、用模板做镂空或裁剪。  
- **核心知识点**：  
  - `blendState` / `depthStencilState` 配置。  
  - 常见 Blend 模式与排序问题。  
  - 使用 Stencil 做 UI 裁剪 / 特效限制区域。  
- **交付物**：若干不同 blend 模式对比 + 一个简单 stencil 裁剪案例。  
- **自检**：切换不同 blend 模式时，前后物体叠加效果可解释且稳定。

#### 第 11 节：性能与移动端适配
- **目标效果**：在移动设备上跑一组前面做过的 Shader，确保帧率和视觉质量可接受。  
- **核心知识点**：  
  - 精度（`highp/mediump/lowp`）选择。  
  - 减少纹理采样次数与分支。  
  - varyings 数量控制，与带宽/性能关系。  
- **交付物**：性能调优前后对比数据（FPS / Profile 截图）。  
- **自检**：在目标机型上长时间运行无明显掉帧/发热异常。

#### 第 12 节：工程化整合与特效库
- **目标效果**：把前面实现的效果整理成「可复用特效库」，用于项目内统一管理。  
- **核心知识点**：  
  - Effect/Material 命名规范与目录结构。  
  - 通过 TS 封装通用组件（如统一时间驱动、统一参数面板）。  
  - 不同项目/关卡间的重用策略。  
- **交付物**：一个包含若干特效的 Demo Scene + 代码仓库结构。  
- **自检**：新增一个特效时，是否能自然融入现有结构而不混乱。
